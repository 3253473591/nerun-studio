# Nerun 工作室网站 · 内容管理手册

## 主要功能

- 展示所有成员的头像、名字、擅长领域（作曲/编曲/调校等）
- 按部门筛选（音乐部、视频部、美术部）
- 点击头像跳转到成员的 B站/网易云主页
- 微信里访问时，会友好提醒"即将跳转到外部网站"

---

## 文件分工说明

### 1. `members.json` —— **成员花名册**
**作用**：存放所有成员的个人资料。

**里面有**：
- 每个成员的姓名、头像照片路径
- 擅长技能（如 `"lyrics"` 代表作词，`"tuning"` 代表调校）
- 使用的软件（如 ACE Studio、Synthesizer V）
- 个人简介（支持插入 B站视频链接）
- 主页链接（B站、网易云等）

**示例条目**（眠铃的档案）：
```json
{
  "id": "Mian Ling",
  "name": "眠铃Neruno",
  "roleIds": ["lyrics", "tuning", "harmony"],  // 作词、调校、和声编写
  "software": "sv",  // 主要用 Synthesizer V
  "avatar": "images/members/眠铃.jpg",  // 头像照片位置
  "bio": "作词/调校/和声编写，工作室负责人...",
  "sortOrder": 0,  // 排序号，0 表示排在最前面
  "isBoss": true,  // 标记为负责人，会显示特殊边框
  "homepages": [   // 点击头像会跳转到这些链接
    {"name": "哔哩哔哩", "url": "https://space.bilibili.com/..."},
    {"name": "网易云音乐", "url": "https://music.163.com/..."}
  ]
}
```

### 2. `roles.json` —— **部门架构图**
**作用**：定义有哪些部门、职位、软件分类，就像公司的组织架构图。

**结构**：
- **音乐部**：编曲、作曲、混音、调校、作词、和声编写
  - 调校角色下还细分软件引擎：ACE Studio、Synthesizer V、Vocaloid 等
- **视频部**：混剪、PV制作、AE特效、字幕特效
- **美术部**：立绘/插画

**每个职位的颜色配置**：
- 编曲是紫色 (`#BB9AF7`)
- 作曲是橙色 (`#E0AF68`)
- 调校是粉红色 (`#F38BA8`)

### 3. `site-config.json` —— **工作室名片**
**作用**：网站顶部的标题、联系方式等公共信息。

**可修改内容**：
- 工作室名称（默认：Nerun工作室）
- 标语（默认：以歌声合成为主的音乐全案制作团队）
- 微信号、邮箱（点击页面顶部按钮可直接复制）
- 负责人特殊标记的颜色（金色边框）

### 4. `ui-text.json` —— **提示语标语牌**
**作用**：页面上的各种提示文字，像展厅里的指示牌。

**包含**：
- "加载失败"时的提示
- "微信号已复制"的提示
- 微信内打开外链时的提示语："即将离开微信..."

### 5. `whitelist.json` —— **可信网站名单**
**作用**：微信内点击链接时，白名单内的网站（如 B站、qq.com）会直接跳转，其他网站会弹出确认框。

**默认已包含**：B站、淘宝、京东、微博、QQ 等。

## 常用操作指南

### 新增一名成员

**步骤**：
1. 准备一张正方形头像照片，放入 `images/members/` 文件夹，命名为成员英文名（如 `zhang-san.jpg`）
2. 打开 `members.json`，在列表最后添加新条目（注意在前后加上英文逗号 `,`）
3. 填写信息：
   - `id`：英文代号（如 `"Zhang San"`），不能重复
   - `name`：显示的中文名（如 `"张三"`)
   - `roleIds`：从 `roles.json` 里找对应的职位代号填入
   - `avatar`：图片路径，如 `"images/members/zhang-san.jpg"`
   - `sortOrder`：数字越小排越前，建议新成员往后排（给大数字）

**格式示例**：
```json
{
  "id": "Zhang San",
  "name": "张三",
  "roleIds": ["composition", "arrangement"],  // 会作曲和编曲
  "avatar": "images/members/zhang-san.jpg",
  "bio": "擅长流行风格作曲",
  "sortOrder": 15,  // 排在第15位
  "homepages": [
    {"name": "哔哩哔哩", "url": "https://space.bilibili.com/123456"}
  ]
}
```

### 修改成员信息

直接在 `members.json` 中找到对应名字，修改相应文字即可。

**特别提示 - 个人简介加链接**：
如果想在简介里插入 B站视频链接（如代表作），可以用这种格式：
```
代表作：[视频标题](https://www.bilibili.com/video/BVxxxxx)
```
显示效果会是一个可点击的蓝色链接。

### 调整排序

修改 `sortOrder` 的数字：
- **负责人**：设为 `0`（最前面，且有特殊金色边框）
- **其他成员**：按加入时间或重要性，数字越小越靠前

### 更换联系方式

修改 `site-config.json` 中的：
- `contact.wechat.id`：微信号（NeurnOfficial）
- `contact.email.address`：邮箱地址（neruuu@qq.com）

修改后，页面顶部的"微信""邮箱"按钮点击后会自动复制新号码。

### 增减部门或职位

修改 `roles.json`：
- **新增部门**：复制一个现有部门块，修改 `id`、`name` 和 `roles` 列表
- **新增职位**：在对应部门的 `roles` 数组中添加新条目，选择一个颜色代码（如 `"#FF6B6B"`）

## 重要提醒

### 1. 图片要求
- **尺寸**：建议 400×400 像素以上的正方形
- **格式**：JPG 或 PNG
- **存放位置**：`images/members/`

### 2. 链接格式
- 必须以 `https://` 开头（如 `https://space.bilibili.com/...`）
- 不要遗漏末尾的斜杠或参数

### 3. 标点符号规则（极易出错！）
- **所有符号必须是英文半角**：`" "` 双引号、`,` 逗号、`[ ]` 方括号、`{ }` 花括号
- **每行末尾的逗号**：除了最后一个条目，每个成员块后面都要加英文逗号 `,`
- **引号成对出现**：`"name": "张三"`，前后引号必须完整

### 4. 备份习惯
修改任何文件前，先复制一份作为备份（如 `members.json.bak`），改错了可以恢复。

### 5. 实时生效
修改保存后，刷新网页即可看到变化（可能需要清除浏览器缓存或等待 5-10 分钟）。

## 🆘 常见问题

**Q：修改后网站显示空白或报错？**
A：通常是标点符号问题。检查：
1. 是否用了中文逗号 `，`（应该用英文 `,`）
2. 是否遗漏了引号或括号
3. 最后一个条目后面是否加了逗号（最后一个不应该加）

**Q：头像显示不出来（显示问号）？**
A：检查 `avatar` 路径是否正确，确认图片确实上传到了 `images/members/` 文件夹，且文件名大小写完全一致（如 `MianLing.jpg` ≠ `mianling.jpg`）。

**Q：新增的成员不显示在页面上？**
A：检查 `roleIds` 是否写对了（如 `"tuning"` 不是 `"tunig"`），且是否遗漏了必需的 `id` 或 `name` 字段。

**Q：点击头像没有反应？**
A：检查 `homepages` 部分的格式，确保 `url` 以 `https://` 开头，且链接有效。

## 📱 微信内访问特别提示

由于微信内置浏览器的安全限制，点击外链（如 B站、网易云）时：
- **白名单内的网站**（B站、qq.com等）：直接跳转
- **其他网站**：会弹出提示框，询问是否继续访问，并提供"复制链接"按钮

这是正常保护机制，不是网站故障。
